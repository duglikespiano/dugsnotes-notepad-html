HTML <template> Element

The <template> element is used to store HTML that should not be rendered immediately. Instead, JavaScript can copy (clone) its contents whenever needed.

Why use <template>?

Normally every HTML element becomes part of the page as soon as the browser parses it.

Sometimes you want HTML that acts as a blueprint instead of being displayed. For example:

  • Chat messages
  • User profile cards
  • Todo list items
  • Product cards
  • Comment blocks
  • Table rows

Instead of writing the same HTML repeatedly with JavaScript, you write it once inside a <template>.

How it works

Everything inside a template is parsed by the browser, but it is stored inside a special object called template.content.

Because it is not part of the rendered document:

  • Nothing is displayed.
  • Images are not rendered.
  • Scripts inside it do not execute.
  • CSS does not affect it.
  • Layout space is not reserved.

Simple Template Example

const template = document.querySelector("#cardTemplate");

const clone = template.content.cloneNode(true);

document.querySelector("#cards").appendChild(clone);

Creating Multiple Elements

template.content

Notice something important:

const template = document.querySelector("#userTemplate");

console.log(template.content);

template.content is a DocumentFragment. It contains all of the HTML inside the template.

You clone this fragment whenever you need a new copy.

cloneNode(true)

Usually you clone the template like this:

const clone = template.content.cloneNode(true);

The true means:

  • Deep clone
  • Copy every child element
  • Copy nested HTML

If you used false, only the root fragment would be copied, not its children.

<template> vs Hidden Elements

Feature <template> Hidden <div>
(display:none)
Visible on page No No
Part of rendered layout No No
Parsed into active DOM No (stored in template.content) Yes
Accessible with normal selectors No (must use template.content) Yes
Consumes rendering work Very little Yes
Scripts inside execute immediately No Yes
Images start loading immediately No Usually yes
Purpose Reusable HTML blueprint Hide existing DOM

Advantages

  • Cleaner JavaScript
  • Reusable UI components
  • No duplicated HTML strings
  • Easy to maintain
  • Safer than building HTML with string concatenation
  • Works well with loops
  • Very fast for repeated UI creation

Disadvantages

  • Requires JavaScript
  • Cannot display itself
  • You must clone it before using it

Browser Support

The <template> element is supported by all modern browsers, including Chrome, Edge, Firefox, Safari, and Opera.

Internet Explorer does not fully support the HTML Template element.

Best Practices

  • Use templates for repeated UI.
  • Keep one template for one component.
  • Update cloned content before inserting it.
  • Avoid constructing long HTML strings with JavaScript.
  • Combine templates with loops for dynamic rendering.
Open Developer Tools (F12) and inspect the page. You'll notice that the template itself never appears in the rendered DOM until JavaScript clones it.
