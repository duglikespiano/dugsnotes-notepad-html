The "popup" HTML Tag

There is no <popup> HTML element in the HTML specification. Many developers mistakenly call the new Popover API a "popup tag", but the actual syntax is the popover attribute.

Comparison Table

Feature Popup (Popover API) <dialog>
Actual HTML tag? ❌ No. Uses an attribute (popover). ✅ Yes.
HTML Syntax <div popover> <dialog>
Primary Purpose Tooltips, menus, small popups Forms, confirmations, modal windows
Modal Support No Yes (showModal())
Backdrop No Yes
Blocks page interaction No Yes (when modal)
Keyboard focus management Minimal Automatic
Close with ESC Usually yes Yes
Best use case Dropdown menus, help bubbles, quick information Login forms, confirmation dialogs, settings windows
Requires JavaScript? No (basic usage) Yes (typically)

Example 2 — Dialog Element

Dialog Window

The <dialog> element is intended for modal windows, forms, confirmations, and other interactions that should temporarily take over the page.

When Should You Use Each?

Use Popover When:

  • Creating dropdown menus
  • Showing tooltips
  • Displaying contextual help
  • Making lightweight popups

Use Dialog When:

  • Building login windows
  • Showing confirmation dialogs
  • Creating forms
  • Displaying settings panels
  • Blocking user interaction until a choice is made

Summary

Incorrect: 

<popup>
    Hello
</popup>

There is no such HTML element.

Correct (Popover API): 

<div popover>
    Hello
</div>

Correct (Dialog): 

<dialog>
    Hello
</dialog>
