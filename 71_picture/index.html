HTML <picture> Element

The <picture> element allows the browser to choose the most appropriate image from multiple sources.

Unlike a normal <img>, a <picture> can provide:

  • Different images for different screen sizes (Responsive Images)
  • Modern image formats (AVIF, WebP) with JPEG fallback
  • Different image crops for mobile and desktop (Art Direction)
Important: The browser checks each <source> from top to bottom. The first matching source is used. If none match, the final <img> is displayed.

Basic Syntax

<picture>
    <source srcset="image.avif" type="image/avif">
    <source srcset="image.webp" type="image/webp">
    <img src="image.jpg" alt="Example">
</picture>

The <img> is always required because it acts as the fallback image. Older browsers that don't understand <picture> will simply display the <img>.

Example 1 — Image Format Fallback

Modern browsers support AVIF and WebP, which are usually much smaller than JPEG.

<picture>
    <source srcset="mountain.avif" type="image/avif">
    <source srcset="mountain.webp" type="image/webp">
    <img src="mountain.jpg" alt="Mountain">
</picture>

This demo uses placeholder images because local files are not included.

Example 2 — Responsive Images

Different images can be used depending on screen width.

<picture>

    <source
        media="(max-width:600px)"
        srcset="mobile.jpg">

    <source
        media="(max-width:1000px)"
        srcset="tablet.jpg">

    <img
        src="desktop.jpg"
        alt="Landscape">

</picture>

Resize your browser window. Although these placeholder images are similar, the browser will actually choose different image files.

Example 3 — Art Direction

Sometimes you don't just want a smaller image—you want an entirely different crop.

Desktop Mobile
Wide landscape photo Tightly cropped portrait photo
Shows entire scenery Focuses on the main subject 
<picture>

    <source
        media="(max-width:700px)"
        srcset="portrait.jpg">

    <img
        src="landscape.jpg"
        alt="Nature">

</picture>

Common Attributes

Element Attribute Purpose
<source> srcset Image file to use
<source> media Media query (screen width, etc.)
<source> type MIME type (image/webp, image/avif)
<img> src Fallback image
<img> alt Accessibility description
<img> loading Lazy loading
<img> width / height Reserve layout space to reduce layout shifts

When Should You Use <picture>?

Use <picture> when:
  • ✔ You want to serve AVIF/WebP with JPEG fallback.
  • ✔ Mobile and desktop should use different image crops.
  • ✔ You want different images for different screen sizes.
  • ✔ Performance matters.
A normal <img> with srcset and sizes is often enough when only the image resolution changes. Use <picture> when you need format selection (such as AVIF or WebP) or completely different image sources based on media queries.

Browser Support

The <picture> element is supported by all modern browsers, including Chrome, Edge, Firefox, Safari, and modern mobile browsers. Support has been widely available since around 2016.

The browser can only choose among image formats that it supports. For example, if AVIF is unsupported, it automatically falls back to the next matching <source> (such as WebP), and finally to the <img> fallback.

Best Practices

  • Always include an <img> fallback.
  • Always provide meaningful alt text.
  • Put the most preferred image format (AVIF) first.
  • Provide WebP before JPEG when possible.
  • Use loading="lazy" for images below the fold.
  • Specify width and height on the <img> to reduce layout shifts.
  • Use <picture> only when multiple sources are actually needed; a simple <img> is easier to maintain for single-image cases.
