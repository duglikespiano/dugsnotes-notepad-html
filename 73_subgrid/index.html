CSS subgrid

What is subgrid?

Normally, when you create a grid inside another grid, the child grid creates its own independent columns and rows.

subgrid lets the child grid reuse the parent's grid tracks instead of defining new ones.

This is especially useful when multiple components need their content to line up perfectly.

Syntax

.parent{
    display:grid;
    grid-template-columns: 1fr 2fr 100px;
}

.child{
    display:grid;

    /* Inherit parent's columns */
    grid-template-columns: subgrid;

    /* Span all parent columns */
    grid-column: span 3;
}

When should you use subgrid?

  • ✔ Product cards
  • ✔ Pricing tables
  • ✔ Dashboards
  • ✔ Profile cards
  • ✔ Calendars
  • ✔ News/article layouts
  • ✔ Forms with aligned labels
  • ✔ Complex nested grid layouts

Browser Support

✅ Current support is very good. Modern versions of Chrome, Edge, Firefox, and Safari support subgrid. It has become practical for production use in most modern web applications.

Things to know

  • A child must already be a display: grid element before using subgrid.
  • The child only inherits the parent tracks (columns and/or rows). It does not inherit the parent's grid items or placement.
  • To inherit columns, use grid-template-columns: subgrid. To inherit rows, use grid-template-rows: subgrid. You can inherit one, the other, or both.
  • The child should span the relevant parent tracks (for example, grid-column: span 3;) so the inherited grid has tracks to work with.
  • For older browsers that do not support subgrid, provide a fallback by defining explicit grid-template-columns values instead.

Summary

subgrid solves one of CSS Grid's biggest limitations: keeping nested grids perfectly aligned.

Instead of redefining column sizes over and over, a nested grid can simply inherit the parent's grid tracks, making layouts easier to maintain and far more consistent.
