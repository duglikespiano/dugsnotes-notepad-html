The HTML inert Attribute

The inert attribute makes an element and everything inside it completely non-interactive.

  • Cannot receive mouse clicks.
  • Cannot receive keyboard focus.
  • Cannot be reached with the Tab key.
  • Input fields cannot be edited.
  • Buttons cannot be pressed.
  • Links cannot be activated.
  • Screen readers generally ignore the subtree.

How it works

Initially, the panel is fully interactive.

When JavaScript adds the inert attribute: 

panel.setAttribute("inert", "");

The browser automatically disables all interaction inside the panel. No extra JavaScript is required.

Removing it restores everything:

panel.removeAttribute("inert");

Common Use Cases

  • Disable the page behind a modal dialog.
  • Prevent interaction while data is loading.
  • Temporarily disable a sidebar.
  • Make a wizard step inactive until the previous step is complete.
  • Disable unfinished UI sections without hiding them.

Browser Support

The inert attribute is now supported by all major modern browsers, including Chrome, Edge, Firefox, and Safari.

Historically, browsers required a JavaScript polyfill because inert was experimental. If you need to support very old browsers, you may still need a polyfill.
