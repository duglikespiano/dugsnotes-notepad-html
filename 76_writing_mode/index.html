CSS writing-mode

What does writing-mode do?

It changes how text flows. Instead of writing text left-to-right, you can make it flow vertically, just like traditional Japanese books or magazine side labels.

The property changes the block flow direction, not merely the text direction.

Comparison of every writing-mode

horizontal-tb

This is the default writing mode. Text goes left to right. New lines appear underneath the previous line. 
writing-mode: horizontal-tb;

vertical-rl

This is vertical text. Columns begin on the right side. This is commonly used for Japanese books. 
writing-mode: vertical-rl;

vertical-lr

This is vertical text. Columns begin on the left side instead. Notice how the column order changes. 
writing-mode: vertical-lr;

sideways-rl

Instead of rotating each character, the whole line is rotated sideways. Browser support is still limited. 
writing-mode: sideways-rl;

sideways-lr

This value has very limited browser support. Many browsers will ignore it or fall back.
writing-mode: sideways-lr;

When should you use writing-mode?

  • Japanese, Chinese, or Mongolian vertical writing
  • Magazine-style layouts
  • Vertical navigation labels
  • Ribbon badges
  • Timeline labels
  • Creative UI components

writing-mode vs transform: rotate()

writing-mode transform: rotate()
Changes how text is laid out. Only rotates the rendered element.
Selection and cursor behave naturally. Text selection can feel awkward.
Line wrapping still works. Layout remains horizontal before rotation.
Best for actual vertical writing. Best for decorative rotation.

Summary

  • horizontal-tb → Normal web pages.
  • vertical-rl → Traditional Japanese vertical writing.
  • vertical-lr → Vertical writing starting from the left.
  • sideways-rl → Rotates whole lines sideways (limited support).
  • sideways-lr → Experimental with poor support.
  • Use writing-mode when the document should actually flow vertically, not simply appear rotated.
