HTML <input> and <datalist>

The <input> element is one of the most commonly used HTML form controls. It allows users to enter data such as text, numbers, dates, emails, passwords, colors, files, and much more.

The <datalist> element provides a list of suggested values for an <input>. Unlike a traditional dropdown (<select>), users can either choose one of the suggestions or type their own custom value.

Basic <input> Example

<input type="text">
<input type="email">
<input type="number">
<input type="date">
<input type="color">
<input type="password">

Basic <datalist> Example

Start typing inside the textbox. The browser will suggest matching values.

<input list="languages">

<datalist id="languages">
    <option value="JavaScript">
    <option value="Python">
    <option value="Rust">
</datalist>
The user may select one of the suggestions OR type a completely different value. The datalist does not restrict the user's input.

Difference Between <datalist> and <select>

Feature <datalist> <select>
User can type custom text ✔ Yes ✘ No
Provides suggestions ✔ Yes ✔ Yes
Restricts to predefined options ✘ No ✔ Yes
Autocomplete experience ✔ Yes ✘ Usually No
Best for Suggestions Fixed choices

How <input> Connects to <datalist>

The connection happens through the list attribute. 

<input list="cities">

<datalist id="cities">
    ...
</datalist>

The value of the list attribute must exactly match the id of the datalist.

Important Notes

  • A <datalist> is invisible until connected to an <input>.
  • Only inputs that support text entry (such as text, search, url, email, tel, and some others) can effectively use a datalist.
  • The browser controls how suggestions are displayed. You cannot fully customize the dropdown using CSS.
  • Users can ignore every suggestion and type their own value.
  • If you need users to select only predefined options, use <select> instead.

Browser Support

The <input> element is universally supported across all modern browsers.

The <datalist> element is supported by all major modern browsers (Chrome, Edge, Firefox, Safari, and Opera), but there are a few limitations:

  • Its appearance varies between browsers and operating systems.
  • You cannot reliably style the suggestion dropdown with CSS.
  • Accessibility support has historically been inconsistent in some browsers and screen readers.
  • Some older browsers either do not support <datalist> or provide only partial support.

For maximum compatibility and complete control over styling and behavior, many websites use JavaScript autocomplete libraries instead of the native <datalist>.
