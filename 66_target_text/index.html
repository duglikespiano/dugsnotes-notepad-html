CSS ::target-text

The ::target-text pseudo-element allows you to style the specific text matched by a Text Fragment URL. Unlike :target, which styles an entire element, this pseudo-element styles only the matched text inside the document.

Browser Support

⚠ Experimental Feature

This pseudo-element is not fully supported across all browsers. At the time of writing:

  • ✅ Google Chrome
  • ✅ Microsoft Edge
  • ✅ Opera
  • ⚠ Firefox (Not supported)
  • ⚠ Safari (Limited / Not supported)

If your browser doesn't support it, everything on this page will still work normally. The browser will simply ignore the ::target-text rule. This is called progressive enhancement.

What is a Text Fragment?

A Text Fragment lets you create a URL that points directly to a piece of text instead of an element ID.

For example:

page.html#:~:text=CSS%20is%20awesome

When the browser opens this URL, it:

  1. Searches the page for the specified text.
  2. Automatically scrolls to that text.
  3. Highlights the matching text.
  4. Applies your ::target-text styles if supported.

Usage Example

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Vestibulum feugiat nunc vel mauris egestas pellentesque. Aliquam erat volutpat.

Learning CSS is awesome because modern CSS contains many powerful features that previously required JavaScript.

CSS is awesome and can create animations, responsive layouts, complex shapes, masks, gradients, and much more.

The phrase "CSS is awesome" appears above so you can target it using a Text Fragment URL.

How to Test

  1. Save this file as target-text-demo.html.
  2. Open it in Google Chrome or another Chromium-based browser.
  3. Locate the phrase "CSS is awesome" in the example above.
  4. Select only those words.
  5. Right-click the selected text and choose "Copy Link to Highlight".
  6. Paste that generated URL into a new browser tab.
  7. The browser should automatically scroll to the text and apply the yellow highlight defined by the ::target-text rule.

Alternatively, you can manually append something similar to the page URL:

#:~:text=CSS%20is%20awesome

CSS Used

::target-text {
    background: gold;
    color: black;
    font-weight: bold;
    padding: 2px 4px;
    border-radius: 4px;
}

If your browser supports ::target-text, the targeted text will receive these styles. Unsupported browsers simply ignore this CSS rule without causing any errors.

When Should You Use It?

  • Highlighting search results shared by URL.
  • Documentation websites linking directly to a sentence.
  • Educational materials referencing a specific paragraph.
  • Knowledge bases with deep links into long articles.
  • Improving the readability of shared links.

Key Takeaways

  • ::target-text styles only the matched text.
  • It works together with Text Fragment URLs.
  • It is different from :target, which styles an entire element.
  • It is different from ::selection, which only works while the user is actively selecting text.
  • Browser support is currently limited, so it should be considered a progressive enhancement.
