CSS content-visibility: auto

A CSS performance feature that allows the browser to skip rendering off-screen content until it becomes visible.

What is content-visibility: auto?

content-visibility is a CSS property, not an HTML tag. It tells the browser whether an element's contents should be rendered.

When using content-visibility: auto;, the browser skips layout and painting for elements that are outside the viewport, improving performance on long pages. 

.card{
    content-visibility:auto;
    contain-intrinsic-size:500px;
}

Why use contain-intrinsic-size?

When the browser skips rendering an element, it doesn't know its actual height yet.

The contain-intrinsic-size property reserves placeholder space until the element is rendered, preventing layout shifts while scrolling.

Comparison: content-visibility: auto vs display: none

Feature content-visibility: auto display: none
Element exists in the DOM ✅ Yes ✅ Yes
Participates in page layout ✅ Yes ❌ No
Occupies space on the page ✅ Yes ❌ No
Browser may skip rendering ✅ Yes (when off-screen) ✅ Yes (always hidden)
Automatically becomes visible while scrolling ✅ Yes ❌ No
Receives pointer events Only when rendered ❌ No
Accessible to JavaScript ✅ Yes ✅ Yes
Best use case Performance optimization for long pages Hide elements completely
Typical use Blogs, feeds, documentation, galleries Menus, dialogs, conditional UI

When should I use it?

  • ✅ Long blog posts
  • ✅ Documentation websites
  • ✅ Product catalogs
  • ✅ Photo galleries
  • ✅ Infinite scrolling feeds
  • ✅ Chat history
  • ✅ Large dashboards

Avoid using it for above-the-fold content or small interface elements that should always be immediately available.
