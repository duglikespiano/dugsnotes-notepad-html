CSS scrollbar-gutter & scrollbar-width

A practical guide with examples and browser support notes.

What is scrollbar-gutter?

Normally, when a scrollbar appears, it consumes part of the content area, which may cause text or centered elements to shift.

scrollbar-gutter allows browsers to reserve space for the scrollbar before it appears, preventing unexpected layout shifts. 

.container{
    overflow:auto;
    scrollbar-gutter:stable;
}
Browser support

Modern versions of Chrome, Edge, Firefox, and Safari support scrollbar-gutter. Older browsers simply ignore the property, so your layout continues to function normally.

What is scrollbar-width?

scrollbar-width controls how thick a scrollbar should be. Unlike scrollbar-gutter, it changes the appearance of the scrollbar rather than reserving layout space. 

.container{
    scrollbar-width:thin;
}
Value Description
auto Default scrollbar width.
thin Uses a thinner scrollbar.
none Hides the scrollbar while still allowing scrolling.
Modern browsers support scrollbar-width, although it originated in Firefox. Older browsers may ignore the property.

Why these two boxes may look identical

If you are using macOS, you may not see any difference at all. This is expected behavior.

By default, macOS uses overlay scrollbars, meaning the scrollbar is painted on top of the content instead of taking up layout space. Since no layout space is consumed, there is nothing for scrollbar-gutter to reserve.

Even on operating systems that use traditional scrollbars, this example can still appear nearly identical because both boxes have a fixed width. The scrollbar appears inside the scrollable element, so the overall box size does not visibly change.

On macOS you can test the property more easily by changing the system setting:

System Settings
→ Appearance
→ Show scroll bars
→ Always

When scrollbars are always visible, browsers reserve physical space for them instead of overlaying them, making the effect of scrollbar-gutter much easier to observe.

Real-world usage

The most common use of scrollbar-gutter is not inside small scrollable elements like these boxes.

Instead, it is typically applied to the entire page to prevent the page from "jumping" horizontally when a modal dialog opens and page scrolling is disabled. 

html{
    scrollbar-gutter:stable;
}

Without this property, hiding the page scrollbar changes the viewport's available width, causing centered layouts to shift horizontally. With scrollbar-gutter: stable, the scrollbar space is always reserved, so the layout remains perfectly stable.

This page-layout stabilization is the primary reason scrollbar-gutter was added to CSS.

Example: scrollbar-gutter: stable both-edges

The both-edges keyword reserves equal space on both sides of the scrolling area. This helps keep centered content visually centered, even when a scrollbar appears. 

.container{
    overflow:auto;
    scrollbar-gutter:stable both-edges;
}

Example: scrollbar-width

.default{
    scrollbar-width:auto;
}

.thin{
    scrollbar-width:thin;
}

.hidden{
    scrollbar-width:none;
}

Best Practices

  • Use scrollbar-gutter to prevent layout shifts.
  • Use scrollbar-width only to adjust scrollbar appearance.
  • Avoid hiding scrollbars for important content because users may not realize that an area is scrollable.
  • Remember that operating systems using overlay scrollbars (such as the default configuration on macOS) may show little or no visible effect from scrollbar-gutter.
